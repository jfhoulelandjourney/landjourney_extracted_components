# landjourney_extracted_components

A standalone Angular 21 + Storybook workspace that mirrors the **real** components from the source app at `lanjourney_local_environement_2026/user-interfaces`, with mock data so each component renders in isolation. Used to browse, redesign, and iterate on the design system without running the full app.

## Stack

- **Angular 21** (standalone components, OnPush, signals)
- **Storybook 10** (`@storybook/angular`)
- **Angular Material 21** + custom SCSS design tokens (copied verbatim from source)
- **NgRx Signals** for stores, **TanStack Angular Table** for grids

## Install & run

```bash
npm install
npm run storybook    # http://localhost:6006
```

## Layout

```
projects/extracted/
  src/
    lib/
      assets/             # static images / icons
      autogenerated/      # OpenAPI types (do not edit)
      constants/          # masks, time, timing
      design-system/      # primitives, molecules, organisms (+ docs.mdx)
      directives/         # activate, spacing, etc.
      dynamic-forms/      # dynamic form + editor (data-heavy)
      models/             # domain models
      pipes/              # date-ago, list, safe-html, due-date
      resolvers/, guards/
      services/           # mocked at story level via providers
      stores/             # NgRx Signals stores (mocked at story level)
      types/, utils/
      ui-styles/          # design tokens, themes, mixins (Wave 1 foundation)
      user-guide/
      web-components/     # bulk of the UI library (form, layout, display, ...)
    public-api.ts         # mirrored from source
.storybook → projects/extracted/.storybook/  (main.ts, preview.ts, ...)
scripts/list-components.mjs  # enumeration / progress diff
EXTRACTION_PROGRESS.md      # auto-generated checklist of every component
```

## Source contract

Components are **copies** of the source files at `lanjourney_local_environement_2026/user-interfaces/projects/common/src/lib/`. The source repo is treated as read-only: edits made here do not flow back. To resync, re-run the copy step in this repo's setup notes.

258 components, 14 directives, 4 pipes were copied during initial extraction. See `EXTRACTION_PROGRESS.md` for the live ticked list as stories are written.

## Adding a story

```ts
// my-thing/my-thing.component.stories.ts
import type { Meta, StoryObj } from '@storybook/angular';
import { MyThingComponent } from './my-thing.component';

const meta: Meta<MyThingComponent> = {
  title: 'Web Components/Display/My Thing',
  component: MyThingComponent,
  tags: ['autodocs'],
};
export default meta;

export const Default: StoryObj<MyThingComponent> = {
  args: { /* mock props */ },
};
```

For components that depend on routing, NgRx stores, or HTTP services, use `applicationConfig({ providers: [...] })` and `moduleMetadata({ imports: [...] })` from `@storybook/angular` to wire mocks per story (see `web-components/button2/button.component.stories.ts` for the canonical pattern).

## Wave plan

1. **Wave 1 — Foundations** (ui-styles, utils, pipes, directives, models, types, autogenerated) ✅
2. **Wave 2 — Primitives & simple form fields** (button2, text/checkbox/select/date inputs, chip, progress-bar, ...)
3. **Wave 3 — Molecules** (avatar, tabs, carousel)
4. **Wave 4 — Low-data organisms** (pagination, side-panel, image-uploader, attachments-list display, ...)
5. **Wave 5 — High-data organisms** (dynamic-form, signature suite, AI checklist, co-applicants, loans, products, businesses)
6. **Wave 6 — App-level shells** (anything from `client-web-app` or `back-office` worth showcasing)
